Modernize the release pipeline, dependencies, and configuration.

### Releases

- Adopt [Changesets](https://github.com/changesets/changesets) for versioning
  and publishing. Each PR now records its intended semver bump and a summary in
  a `.changeset/*.md` file; merging to `main` opens a "Version Packages" PR
  whose merge publishes to npm via OIDC Trusted Publishing.
- The legacy tag-based `publish.yml` workflow has been retired. The
  `release:changelog` / `postversion` scripts and the `conventional-changelog-cli`
  devDependency are gone — the `CHANGELOG.md` is now generated by
  `@changesets/changelog-github` and contains PR/commit links.

### Configuration

- New `~/.easyui5rc.json` (preferred) and `~/.easyui5rc` (INI) for `generator-easy-ui5`-specific options
  (`ghAuthToken`, `ghBaseUrl`, `pluginsHome`, `addGhBaseUrl`, `addGhOrg`, `addSubGeneratorPrefix`). The lookup walks
  from `cwd` up to your home directory (ESLint-style cascade), and a file may
  set `"root": true` to stop the walk.
- New `EASY_UI5_*` environment variables for CI (e.g.
  `EASY_UI5_GH_AUTH_TOKEN`) that override every config file.
- The legacy `easy-ui5_*` keys in `~/.npmrc` are still read for one release,
  with a one-time stderr deprecation notice — npm 11 already warns about them
  and npm 12 will reject them. Move them to `~/.easyui5rc.json`.
- Internals: the config reader no longer shells out to
  `npm config list --json`, removing a child-process dependency and ~100–300 ms
  of startup cost.

### Dependencies and tooling

- Minimum supported Node.js is now **20.19** (was `>=20`) to match
  `eslint@10`'s engines field.
- Bumped `yeoman-generator` to `^8.2.2` and `eslint` to `^10.5.0`.
- Added `eslint-plugin-n` and `eslint-plugin-security` plus a strict CI variant
  (`npm run lint:ci`, `--max-warnings 0`) for supply-chain hygiene.
